# vino 功能清单

> 更新时间：2026-03-25  
> 适用范围：`vino_iPhone`、`vino_Desktop`、控制协议、数据链路  
> 当前阶段：工业检测系统 MVP 向可交付版本推进

## 1. 状态说明

- `已实现`：仓库内已有可运行代码或可编译工程。
- `MVP`：已经有骨架或主链路，但仍需补完边角能力、异常处理或最终 UI。
- `待开发`：需求已明确，但当前仓库尚未真正落地。
- `暂缓`：当前版本不纳入，未来可能进入路线图。

---

## 2. 总体目标

`vino` 要解决的是一套工业检测现场的双端协同问题：

1. iPhone 负责采集、推理、局部交互与边缘输出。
2. Desktop 负责设备发现、集中控制、模型分发、数据终端与外部系统接入。
3. 两端共享同一套控制协议、上下文字段和结果语义。
4. 所有远程触发都要围绕 `productUUID`、`pointIndex`、`jobId` 建立追踪链路。

---

## 3. 范围边界

### 3.1 当前在范围内

### 3.2 当前不在范围内

- Web 前端控制台
