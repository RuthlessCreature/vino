# vino_platform PRD

- Product: `vino_platform`
- Owner: Vino
- Date: 2026-04-29
- Status: Draft

## 问题

工业视觉模型从开发者交付到客户现场，当前通常靠人工传文件、线下合同、手工授权、现场调试。这个流程有四个硬伤：

- 采购方难判断模型质量、适配场景、授权边界和售后责任。
- 开发者难安全分发模型、追踪授权、收款结算和持续维护。
- 平台方难审核模型版权、管控交易风险、沉淀设备和项目数据。
- 现场终端难在离线环境下安全使用已购买模型。

`vino_platform` 要把这件事做成可产品化交付的闭环。

## 用户角色

| 角色 | 说明 | 核心目标 |
| --- | --- | --- |
| 企业采购方 | 工厂、集成商、项目团队管理员 | 找到可用模型，购买授权，分配给现场设备 |
| 现场操作员 | 使用 `vino_iPhone` 的终端用户 | 登录后拿到可用模型并稳定推理 |
| 模型开发者 | 个人、团队、企业算法供应商 | 上架模型，成交，获得结算和售后入口 |
| 平台运营 | 审核、上架、推荐、订单、纠纷处理 | 控制质量、交易、安全和收入 |
| 财务人员 | 平台或开发者财务 | 对账、开票、结算、提现 |
| 合规审核员 | 模型版权、内容、行业合规审核 | 降低侵权、违法和安全风险 |

## 范围

### P0

- 账号与组织：注册/创建、登录、角色权限、组织用户管理。
- 开发者最小入驻：资质资料、协议状态、店铺基础信息。
- 模型管理：上传、格式校验、版本、构建、哈希、分类标签、上下架。
- 模型审核：基础元数据审核、文件审核、版权材料记录、审核意见。
- 模型商城基础：列表、搜索、分类、详情、价格、开发者信息。
- 订单基础：创建订单、线下收款确认、订单状态、退款标记。
- 授权系统：Entitlement、设备绑定、离线租约、续租、撤销。
- 模型下载：短时下载票据、加密包、哈希校验、下载审计。
- 运营后台：用户、开发者、模型、订单、授权、设备、审计日志。
- iPhone 兼容：保留 `vino_cloud` 已有终端接口语义。

### P1

- 在线支付：微信、支付宝、对公转账回单、银行卡。
- 开发者结算：佣金规则、结算单、提现、对账。
- 发票：普通/专用发票申请、开票状态、电子发票。
- 模型试用：云端试用、样例数据、次数限制、试用反馈。
- 售后工单：问题提交、开发者响应、平台介入、退款仲裁。
- 评价体系：采购方评价、评分、案例、复购线索。
- 推荐运营：首页推荐、行业精选、活动、优惠券。

### P2

- 定制需求市场：企业发需求、开发者报价、里程碑验收。
- 算力和数据服务：第三方算力、合规数据集、训练任务。
- 智能推荐：按行业、设备、项目、历史购买推荐模型。
- 私有化部署：企业内网部署、专属对象存储、专属密钥。

### Non-goals

- 本轮不修改 `vino_iPhone`。
- 不在 `vino_iPhone` 内做完整购买、支付、促销和开发者中心。
- 不做通用插件、小程序或脚本市场。
- 不把首发市场定位成面向消费者的 AI 内容商店。
- P0 不要求自动化税务、自动分账和复杂推荐。

## 核心流程

### 模型上架

1. 开发者入驻并签署协议。
2. 上传模型文件和版权/训练来源材料。
3. 系统生成 `ModelBuild`，计算哈希，识别格式与大小。
4. 开发者填写分类、场景、精度、输入输出、部署说明、价格。
5. 平台审核通过后模型进入可售状态。

### 购买授权

1. 采购方选择模型 SKU。
2. 生成订单并选择授权方式：永久、年费、项目期、试用。
3. P0 由平台运营确认线下收款；P1 接入在线支付。
4. 支付成功后生成 Entitlement。
5. 管理员把授权分配给组织、用户或设备。

### iPhone 下载

1. 终端登录平台账号。
2. 终端请求可用模型列表。
3. 平台只返回当前账号可用 Entitlement 对应模型。
4. 终端申请短时下载票据。
5. 平台生成设备绑定租约和加密下载地址。
6. 终端下载、解密、校验哈希、安装模型。

### 离线续租

1. 终端启动或运行前校验本地租约。
2. 租约有效则允许推理。
3. 租约即将过期则提示联网续期。
4. 合同过期、授权撤销或设备解绑后，不再发放新租约。

## UX 原则

- 采购方看到的是“能解决哪个检测场景”，不是模型参数堆砌。
- 开发者上传流程必须少字段启动，审核前再补充完整材料。
- 运营后台要优先可扫读：风险状态、订单状态、授权状态、下载状态必须直接可见。
- 终端侧只出现必要动作：登录、同步、下载、续租、启用。
- 错误必须可恢复：下载失败可重试，哈希失败要重发票据，授权过期要指向管理员。

## 指标

| 类别 | 指标 |
| --- | --- |
| Activation | 首个企业从创建到 iPhone 成功下载模型的用时 |
| Engagement | 每周模型下载次数、活跃设备数、续租次数 |
| Revenue | 订单金额、付费组织数、模型复购率、开发者成交数 |
| Quality | 下载成功率、授权误放率、审核退回率、售后工单率 |
| Security | 未授权访问拦截数、过期票据请求数、异常下载告警数 |

## 发布标准

Product:

- P0 角色和流程可完整演示。
- 采购方、开发者、运营后台的核心状态清晰。
- iPhone 只看到已授权模型。

Engineering:

- API 有版本号、认证、错误码、幂等键。
- 模型文件有对象存储记录、哈希、版本和审计。
- 下载票据短时有效，过期不可复用。

Quality:

- 至少覆盖账号、模型、订单、授权、下载、续租的端到端测试。
- 未授权、过期、设备不匹配、哈希失败等负向场景必须通过。

GTM:

- 支持至少 3 类工业视觉模型样例。
- 支持线下合同/线下付款的试点客户交付。
