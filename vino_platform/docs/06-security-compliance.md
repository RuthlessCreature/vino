# vino_platform 安全与合规方案

- Product: `vino_platform`
- Date: 2026-04-29
- Status: Draft

## 安全目标

- 未授权用户不能发现、下载或续租模型。
- 模型下载链接短时有效，不可长期分享。
- 模型文件传输和落地要有加密、哈希和设备绑定。
- 交易、授权、审核、下载、撤销必须可审计。
- 平台不把 `vino_iPhone` 做成消费级 App 内模型商城。

## 账号与权限

P0：

- Web 和终端统一账号体系。
- Bearer token 会话，服务端可撤销。
- RBAC：buyer_admin、buyer_operator、developer_admin、developer_member、platform_ops、reviewer、finance、super_admin。
- 关键操作二次确认：下架、撤销授权、确认收款、退款。

P1：

- MFA。
- 企业 SSO。
- IP allowlist。
- 高危操作审批流。

## 模型资产保护

### 上传侧

- 文件格式白名单：`.mlmodel`、`.mlpackage`、`.mlmodelc`。
- 计算并保存明文包 `sha256`。
- 扫描文件大小、目录结构、Manifest。
- 上传后进入隔离区，审核通过才进入可分发区。

### 下载侧

沿用 `vino_cloud` 的方向：

- 下载前必须校验 Entitlement。
- 生成短时 `DownloadTicket`。
- 票据绑定 user、organization、device、model、modelBuild。
- 服务端返回 `vino-aesgcm-v1` 加密包。
- 终端使用临时 `ticketSecret` 派生密钥。
- 终端解密后校验 `sha256`，再安装。

生产增强：

- 服务端不长期保存 `ticketSecret` 明文，只保存 hash 或加密封装。
- 下载 URL 使用 HTTPS。
- 大文件下载限速、限并发、限地域或限组织策略。
- 异常下载触发告警。

## 授权与离线租约

原则：

- Entitlement 是长期商业权利。
- OfflineLease 是终端短周期运行权利。
- 永久授权也建议使用周期性离线租约。

建议默认：

| 授权类型 | 合同权利 | 离线租约 |
| --- | --- | --- |
| 试用 | 7 到 14 天 | 1 到 7 天 |
| 年费 | 到合同截止日 | 7 到 30 天 |
| 项目期 | 到项目截止日 | 7 到 30 天 |
| 永久 | 指定主版本永久可用 | 30 到 90 天循环续租 |

撤销规则：

- 新下载票据立即停止发放。
- 新续租立即停止发放。
- 已下载模型在本地租约到期后失效。
- 高风险场景可加入紧急吊销名单，终端联网后立即失效。

## 交易安全

P0：

- 线下收款由运营确认，必须上传凭证或备注。
- 订单确认收款后才生成 Entitlement。
- 订单、付款、授权创建写入审计日志。

P1：

- 支付回调必须验证签名。
- 回调按 provider_trade_no 和 idempotency_key 去重。
- 退款必须回写授权策略：撤销、缩短租约或保留历史版本。
- 平台佣金和开发者结算从订单快照计算，不依赖可变价格。

## 知识产权审核

开发者上架必须提供：

- 模型来源声明。
- 训练数据合法性声明。
- 权利归属证明或授权证明。
- 禁止用途声明。
- 如涉及第三方框架或预训练权重，提供 license 信息。

平台审核：

- 审核模型名、描述、样例、使用场景是否夸大或违法。
- 审核是否冒用知名模型、客户数据或第三方商标。
- 侵权投诉进入 `ReviewCase`，处理结果写入 `AuditLog`。

## 内容合规

禁止上架：

- 明确违法违规用途模型。
- 绕过安全、入侵、欺诈、隐私侵犯用途模型。
- 未获授权的人脸识别、身份识别、敏感个人信息处理模型。
- 无法解释来源且风险较高的数据集或权重。

工业视觉白名单优先：

- 缺陷检测。
- 尺寸测量。
- OCR/条码/铭牌识别。
- 设备状态识别。
- 安全帽、工服、区域入侵等合规现场安全检测。

## 隐私与数据

- 用户密码只存 hash。
- token、票据、支付流水、身份证照、营业执照等敏感信息脱敏展示。
- 图片、视频、推理结果按组织隔离。
- ingest 支持客户配置保留期。
- 导出和删除数据要写审计。
- 生产环境必须使用 HTTPS。

## App Store 边界

推荐叙事：

- `vino_iPhone` 是工业现场终端。
- Web 平台是企业模型授权、项目和设备管理后台。
- 购买行为在 Web 或销售合同中完成。
- iPhone 不提供消费级模型商城，不展示促销导购，不处理支付。
- iPhone 下载的是企业授权的受控 CoreML 模型资产，不是通用插件或脚本。

高风险做法：

- 在 iPhone 内展示完整模型商城和外部购买按钮。
- 面向普通消费者售卖 App 内能力解锁。
- 下载可执行脚本、通用插件或动态小程序能力。

## 审计日志

必须记录：

- 登录、失败登录、高危权限变更。
- 开发者入驻审核。
- 模型上传、审核、上架、下架。
- SKU 创建和价格修改。
- 订单创建、确认收款、退款。
- Entitlement 创建、分配、撤销、续期。
- 下载票据发放、下载成功、下载失败。
- 设备绑定、解绑、封禁。

审计字段：

- actor、action、object、before/after 摘要、IP、user agent、requestId、createdAt。
